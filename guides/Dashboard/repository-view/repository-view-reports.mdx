Repository Reports
==================

The Reports tab displays repository-level reports focused on security issues and codebase insights.

Reports: Security

[](#reports-security)
------------------------------------------

### 

OWASP Top 10 & CWE/SANS Top 25

[](#owasp-top-10--cwesans-top-25)

The OWASP Top 10 and CWE/SANS Top 25 reports give you a bird’s eye view of how your repository measures up to these popular security recommendations. You can identify the exact issues that need to be fixed in one click, filtered by severity. You may choose to share these reports with external stakeholders.

![](https://files.readme.io/a59ae1a-owasp-repo.png)

![](https://files.readme.io/c63c639-SANS-repo.png)

Reports: Insights

[](#reports-insights)
------------------------------------------

### 

Code Health Trend

[](#code-health-trend)

The Code Health Trend report provides valuable insights into the number of new code health issues that emerge in your repository every week. By monitoring the net new issues, you can mitigate the risk of facing unforeseen challenges in the future.

![](https://files.readme.io/7d1396a-CHT-repo.png)

### 

Issues Prevented

[](#issues-prevented)

By using the Issues Prevented report, you can gain a historical perspective on the number of code health issues that were prevented from entering your repository with the help of DeepSource. This report can help you quantify the impact of incorporating DeepSource into your software development workflows. Moreover, over time, this report can provide a visual representation of how code health automation affects software quality and delivery.

![](https://files.readme.io/10c80ec-issues_prevented_-_repo.png)

### 

Issues Autofixed

[](#issues-autofixed)

The Issues Autofixed report offers insights into how you leverage DeepSource Autofix™️ to automate the process of cleaning up code health issues in the repository. This report enables you to evaluate the time and effort you save by the automated fixing process.

![](https://files.readme.io/5f90327-issues_autofixed_-_repo.png)

### 

Issue Distribution

[](#issue-distribution)

The Issue Distribution report provides an overview of the current state of code health issues in the repository by issue category and programming language. This report is particularly useful for identifying which aspects of your code health are deteriorating the most. By presenting a historical view of the distribution across various issue categories and languages, you can prioritize actions and address the areas that require immediate attention.

![](https://files.readme.io/03fe6c6-issue_distribution_-_repo.png)

Public Reports

[](#public-reports)
-------------------------------------

You can create customized, password-protected public reports that can be shared with others. These public reports provide a comprehensive summary of the selected report and include information on their intended use.

![](https://files.readme.io/d7e77b0-public_reports_-_repo.png)
