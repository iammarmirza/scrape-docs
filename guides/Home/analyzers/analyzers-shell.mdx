Shell
=====

The Shell Analyzer analyzes your shell scripts and raises issues if something is wrong with them. It points out bugs and syntax issues with a detailed explanation of how to fix them.

This section covers configuration specific to the `Shell` analyzer. Please make sure to read the general [configuration guide](https://deepsource-docs.readme.io/docs/configuration) first.

Configuration - `.deepsource.toml`

[](#configuration---deepsourcetoml)
-------------------------------------------------------------------------

### 

`name`

[](#name)

*   **Type**: [String](https://toml.io/en/v1.0.0#string)
*   **Presence**: mandatory
*   **Description**: Shortcode of the analyzer.
*   **Example**:

TOML

`name = "shell"`

### 

`enabled`

[](#enabled)

*   **Type**: [Boolean](https://toml.io/en/v1.0.0#boolean)
*   **Presence**: optional
*   **Description**: Toggle whether this analyzer should be run.
*   **Example**:

TOML

`enabled = true`

### 

`meta`

[](#meta)

*   **Type**: [Table](https://toml.io/en/v1.0.0#table)
*   **Presence**: optional
*   **Description**: Analyzer specific metadata to tweak its behavior. These values are used to customize the checkers.
*   **Example**:

TOML

`[analyzers.meta]   dialect = "bash"`

Available `meta` field

[](#available-meta-field)
---------------------------------------------------

#### 

`dialect`

[](#dialect)

*   **Type**: [String](https://toml.io/en/v1.0.0#string)
*   **Presence**: optional
*   **Description**: UNIX shell on which the script is supposed to be run.
*   **Available Value**: `sh`, `bash`, `dash` and `ksh`.
*   **Default Value**: The analyzer detects the shell from the file's shell directive or the shebang line by default.
*   **Example**:

TOML

`[analyzers.meta]   dialect = "sh"`

Sample config

[](#sample-config)
-----------------------------------

TOML

`version = 1  [[analyzers]] name = "shell" enabled = true    [analyzers.meta]     dialect = "zsh"`
